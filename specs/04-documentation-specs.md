### Documentation specifications
* Document the project in a README.md file with a clear description of the project, its features, and how to use it
* maintain a clear diagram of the project structure and embedded in the README.md file
* maintain a clear changelog.md file with a list of changes

* Make sure specs are up to date
* Make sure code is documented
* Make sure the documentation is clear and easy to understand
* Make sure the documentation is complete
* Make sure the documentation is easy to find
